# РЕКОМЕНДАЦИИ ПО ИСПОЛЬЗОВАНИЮ И РАЗВИТИЮ ПРОЕКТА

## Текущее состояние

Проект "Admissions_Reserve" - это приложение для управления приёмом заявлений на обучение, построенное на WPF с использованием SQLite.

**Версия**: .NET Framework 4.8  
**Тип приложения**: Desktop (WPF)  
**БД**: SQLite

---

## 📋 СТРУКТУРА ПРИЛОЖЕНИЯ

### Страницы (Pages)

```
1️⃣  IdentityPage           - Данные удостоверения личности
2️⃣  ContactsPage            - Контактная информация
3️⃣  ApplicationTypeAndEducationPage - Тип заявления и образование
4️⃣  DocumentsPage           - Документы абитуриента
5️⃣  RelativesPage           - Информация о родственниках
6️⃣  AdditionalInfoPage      - Дополнительная информация
7️⃣  ApplicationCompetitionsPage - Выбор конкурсов
8️⃣  IndividualAchievementsPage - Индивидуальные достижения
9️⃣  PrioritiesPage          - Приоритеты конкурсов

```

### Модели данных (Model)

- `Models.cs` - Основные сущности (Applicants, Documents, etc.)
- `ValidationHelper.cs` - Вспомогательные функции валидации
- `DataService.cs` - Доступ к БД
- `DatabaseHelper.cs` - Управление подключением к БД
- `SessionManager.cs` - Управление сессией текущего пользователя

## ❓ FAQ

**Q: Как добавить новое поле на странице?**  
A: Добавьте TextBox/ComboBox в XAML, затем привяжите к модели в Code-Behind, добавьте валидацию в методе SaveData()

**Q: Как создать новую страницу?**  
A: Создайте новый файл Page.xaml, скопируйте структуру из существующей страницы, добавьте в ApplicantWizardPage

**Q: Как добавить новый тип документа?**  
A: Добавьте в таблицу DocumentTypes БД, затем загружайте через DataService.GetAll<DocumentTypes>()

**Q: Что делает SessionManager?**  
A: Хранит текущего пользователя (абитуриента) в памяти между страницами

**Q: Почему все страницы проверяют SessionManager?**  
A: Чтобы пользователь не мог перейти на страницу без создания абитуриента
